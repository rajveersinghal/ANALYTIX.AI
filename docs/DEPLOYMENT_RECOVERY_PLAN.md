# Deployment & Login Resolution Plan

The current login failures on Vercel are likely caused by the backend (Python Serverless Functions) failing to deploy due to the 500MB size limit of Vercel's standard lambda functions (caused by heavy ML dependencies like `scikit-learn`, `pandas`, and `scipy`).

To fix this, we will transition to a **Decoupled Architecture**:
1. **Frontend**: Hosted on Vercel (Optimized for static assets).
2. **Backend**: Hosted on Render (Docker container with persistent storage and large dependency support).

## Phase 1: Fix Vercel Deployment

Vercel is currently struggling with the `requirements.txt` and `api/index.py`. We will simplify `vercel.json` to treat the project as a pure React application.

### 1.1 Update `vercel.json`
Remove the Python build instructions and strictly handle routing for the React SPA.

```json
{
  "version": 2,
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

### 1.2 Set Environment Variables
In the Vercel Project Dashboard (Settings > Environment Variables), add:
- `VITE_API_URL`: `https://analytix-api.onrender.com` (Replace with your actual Render URL)

## Phase 2: Verify Backend on Render

Ensure the backend is healthy and can communicate with the frontend.

### 2.1 Update CORS in `app/main.py`
Verify that `ALLOWED_ORIGINS` includes the Vercel deployment URL.
- **Render Dashboard**: Set `ALLOWED_ORIGINS` env var to `https://your-app.vercel.app`.

### 2.2 Synchronize `JWT_SECRET`
Ensure the `JWT_SECRET` in Vercel matches the `JWT_SECRET` in Render. If they mismatch, tokens generated by one won't be valid for the other (causing immediate logout or "Unauthorized" errors).

## Phase 3: Validation Checklist

- [ ] **Vercel Build**: Check that the build completes in < 2 mins without installing Python packages.
- [ ] **Network Logs**: Open Browser DevTools (F12) -> Network tab. Try logging in.
    - If `OPTIONS` request fails: Check CORS on Render.
    - If `POST /auth/login` returns 404: `VITE_API_URL` is likely wrong.
    - If `POST /auth/login` returns 500: Check Render logs for database connectivity or `JWT_SECRET` issues.

## Immediate Local Fix (Optional)
If you are testing locally and login is failing, ensure the backend is running:
```bash
python -m uvicorn app.main:app --reload
```
And check that your local `.env` has `VITE_API_URL=http://localhost:8000`.
